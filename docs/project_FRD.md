# 1. System Overview

The Helios Event website will operate as a CMS‑driven platform that allows administrators to manage services, portfolio events, blogs, careers, vendor registrations, and customer inquiries.

The system must support content management, SEO optimization, media management, and lead generation features required for an event management company.

The platform will include both:

• Public Website (User Facing)
• Admin Dashboard (Content Management)

---

# 2. User Roles

## 2.1 Website Visitor

A visitor can:

• View services
• Browse portfolio events
• Read blog articles
• Apply for careers
• Register as a vendor
• Submit event inquiries

## 2.2 Administrator

Admin users can:

• Manage services
• Manage portfolio categories
• Manage portfolio events
• Manage blog posts
• Manage careers
• View job applications
• Manage vendor registrations
• View inquiries
• Manage city and location pages
• Manage media files

# 3. Navigation System

# 4. Services Module

## User Side

Users can:

• View services listing
• Open individual service pages
• View related portfolio events
• Submit inquiry for the service

## Service Page Structure

Each service page must contain:

• Service title
• Description
• Event images
• Related portfolio events
• Inquiry call‑to‑action

## Admin Side

Admin can:

• Add new services
• Edit service content
• Upload service images
• Manage service URL slug
• Manage SEO metadata
• Enable or disable services
